Detailed Description

Container of one or many Option objects, both standard and custom options.

This is an extension to std::vector, providing some interfaces for interacting with the underlying option structure. It contains one or more Option objects.

void askr::Options::add_yaml ( const YAML::Node &  node)

Parse the options section out of the YAML.

This is used for scripts to add additional, specific command line options. The format of such configs must contain the following attributes:

Attribute Description Required Default
name The identifying name of the configuration For plugin options
description A brief description of the option yes
short The short option character, e.g. -s yes
argument Are arguments required or even allowed no none
long The long name of the option, e.g. '–select no
Parameters
nodeThe YAML node for the options section

std::string askr::Options::get_short_options ( )

Get the short options string out of a vector of Options.

This is necessary to produce a configurable short-options string, which must be built up using the information from the long-options. This does produce a std::string copy, but not important for performance obviously.

Returns
